# Publishing Checklist

这个清单是为了把仓库从“个人练习”推进到“别人愿意 star 和转发”的状态。

## 上线前必须补齐

- 补一张仓库封面图或架构图
- README 首页补一段 30 秒价值说明
- 至少提供 5 个可直接复制执行的 SQL 案例
- 补一张 Flink Web UI 或运行结果截图
- 增加英文 README 或英文摘要
- 仓库设置 Topics，例如 `flink`, `flink-sql`, `big-data`, `streaming`, `sql`, `chinese`

## 发布后一周内应该做

- 写 1 篇文章介绍仓库定位和使用方法
- 再写 1 篇文章拆解其中一个高频案例
- 在 README 加上更新日志区块
- 增加一个“适合面试复习”的目录

## 拆分为独立仓库前检查项

- 仓库名确定，例如 `flink-sql-examples-cn`
- 仓库描述一句话说清定位
- 设置 Topics，例如 `flink`, `flink-sql`, `streaming`, `sql`, `big-data`, `chinese`
- 准备一张首图或运行截图
- 准备 social preview，例如 `assets/repo-cover.svg`
- 确认 `Makefile`、`.editorconfig`、`.gitattributes` 一并带出
- 准备首篇介绍文章或发布帖
- 首批至少保留 5 到 8 个能直接展示价值的案例

## 不要做的事情

- 不要一开始就塞 50 个空目录
- 不要只堆技术名词，不给可运行内容
- 不要 README 全是个人介绍，缺少项目价值
- 不要长期只有计划，没有实际案例
